# 使用示例和教程

## 快速开始

### 方法1：使用启动脚本（推荐）

**首次使用：**
1. 双击运行 `install_requirements.bat` 安装依赖包（仅需一次）
2. 等待安装完成

**以后每次启动：**
1. 双击运行 `启动下载器_Fluent.bat` 即可直接启动程序（无控制台窗口）
2. 如需查看报错，运行 `启动下载器_Fluent_调试.bat`

### 方法2：手动启动

```bash
# 1. 首次安装依赖
pip install -r requirements.txt

# 2. 启动GUI程序
python gui_downloader_fluent.py
```

> 当前版本仅提供 GUI 入口，`downloader.py` 为内部下载核心模块。

## 完整使用流程

### 第一步：获取漫画URL

1. 打开浏览器访问零漫画网站
2. 找到想要下载的漫画
3. 复制漫画主页的URL

**URL格式示例：**
```
https://www.zerobywai.com/pc/manga_pc.php?kuid=21019
```

### 第二步：解析漫画

1. 在程序首页的输入框中粘贴URL
2. 点击"解析"按钮
3. 等待解析完成（通常1-2秒）

**解析后会显示：**
- 漫画标题
- 可用章节数量
- 所有章节列表

### 第三步：选择章节

**全选所有章节：**
- 点击"全选"按钮

**选择特定章节：**
1. 点击"全不选"按钮
2. 逐个勾选需要的章节

**批量选择：**
- 按住 Ctrl 键点击可以选择多个章节
- 按住 Shift 键点击可以选择连续的章节

### 第四步：开始下载

1. 点击"开始下载"按钮
2. 程序会自动：
   - 探测每个章节的图片数量
   - 自动识别不同图片格式
   - 并发下载多张图片
   - 自动重试失败的下载
   - 显示实时进度

**下载过程中：**
- 进度条显示总体进度
- 日志区域显示详细信息
- 可以点击"取消下载"中止

### 第五步：查看结果

下载完成后：
1. 程序会弹出完成提示框
2. 显示下载统计信息
3. 漫画保存在配置的下载目录中

**目录结构（输出格式=章节文件夹）：**
```
下载目录/
└── 漫画名称/
    ├── 1/
    │   ├── 0001.jpg
    │   ├── 0002.jpg
    │   └── ...
    ├── 2/
    │   ├── 0001.jpg
    │   └── ...
    └── ...
```

**目录结构（输出格式=ZIP/CBZ）：**
```
下载目录/
└── 漫画名称/
    ├── 第1话.zip/.cbz
    ├── 第2话.zip/.cbz
    └── ...
```

## 高级设置

### 调整下载参数

在"设置"标签页可以调整：

**1. 下载线程数（1-50）**
- 默认：15
- 较小值（5-10）：适合网速较慢或避免被限速
- 较大值（20-30）：适合网速快速下载
- 注意：过大可能导致被网站封IP

**2. 重试次数（1-10）**
- 默认：3
- 网络不稳定时可以增加
- 每次失败后会自动重试

**3. 重试延迟（0-60秒）**
- 默认：2秒
- 两次重试之间的等待时间
- 实际等待时间会指数增长

**4. 请求超时（3-120秒）**
- 默认：15秒
- 适用于连接与读取超时控制
- 网络不稳定时可适当调大

**5. 输出格式（章节文件夹 / ZIP / CBZ）**
- 默认：章节文件夹
- 选择 ZIP/CBZ 时，会输出“漫画文件夹 + 各章节压缩包”
- CBZ 与 ZIP 内容相同，仅文件后缀不同

**6. 下载目录**
- 默认：用户的Downloads文件夹
- 点击"打开"按钮选择其他目录

**7. 图片完整性校验**
- 默认开启“严格校验已存在图片”
- 开启时会检测损坏图片并自动重下
- 关闭时仅按文件大小判断是否跳过

## 常见问题

### Q1: 解析失败怎么办？

**可能原因：**
- URL格式不正确
- 网络连接问题
- 漫画已被删除

**解决方法：**
1. 检查URL是否完整
2. 重新复制漫画页面的URL
3. 确认网络连接正常

### Q2: 下载速度慢？

**优化建议：**
1. 增加下载线程数（设置 → 下载线程数）
2. 检查网络带宽
3. 选择网络较空闲的时段下载

### Q3: 部分图片下载失败？

**可能原因：**
- 网络不稳定
- 服务器临时故障
- 图片确实不存在

**解决方法：**
1. 增加重试次数
2. 增加重试延迟
3. 重新运行下载（默认会校验损坏图片并重下）

### Q4: 部分章节无法下载？

**检查步骤：**
1. 确认URL是漫画主页而不是章节页
2. 确认网络连接正常
3. 尝试增加重试次数
4. 查看日志中的错误提示，可能需要调整下载参数

### Q5: 下载后图片打不开？

**可能原因：**
- 下载中断导致文件损坏
- 磁盘空间不足

**解决方法：**
1. 删除损坏的文件
2. 重新下载该章节
3. 检查磁盘空间

### Q6: 避免服务器负载？

**最佳实践：**
1. 合理设置下载线程数（建议不超过15）
2. 增加重试延迟
3. 分批下载，不要一次下载太多
4. 在网络空闲时段下载

## 技术细节

### 线程安全

程序使用 `ThreadPoolExecutor` 管理并发下载，确保线程安全的同时提高下载效率。

### 重试机制

采用指数退避策略，避免对服务器造成压力，同时提高网络不稳定环境下的下载成功率。

### 进度反馈

使用信号槽机制实现GUI实时更新，用户可以清楚地了解下载进度和状态变化。

## 注意事项

1. **尊重版权**
   - 下载的漫画仅供个人学习使用
   - 不要用于商业用途
   - 建议支持正版

2. **合理使用**
   - 不要同时下载过多漫画
   - 避免对服务器造成过大压力
   - 建议在非高峰时段下载

3. **数据安全**
   - 定期备份下载的漫画
   - 配置文件包含下载偏好，建议妥善保管
   - 不要与他人共享配置文件

4. **法律风险**
   - 使用自行承担风险
   - 部分地区可能有法律限制
   - 建议咨询当地法律
